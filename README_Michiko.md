## ✅ 無理なく進められるおすすめ順序（残り5個）

### ① Using GitHub Copilot CLI

**難易度：★☆☆（かなり低）**

**理由**

*   コードを書く必要がほぼない
*   自然言語 → コマンド生成が中心
*   Copilotの「使う側」に徹できる

**学べること**

*   「何をどう頼むとCopilotが正確に動くか」
*   プログラミング経験が少なくても成功体験を得やすい

👉 **ウォーミングアップ兼、自信回復用として最適**

***

### ② Using Advanced GitHub Copilot Features

**難易度：★★☆**

**理由**

*   inline chat / slash commands / agents が中心
*   既存コードを「読む → 質問する」スタイル
*   新規実装より「理解・確認」がメイン

**学べること**

*   「コードが分からなくてもCopilotに聞けば良い」体験
*   自分が書いた覚えのないコードへの向き合い方

👉 **プログラマー初心者との相性が非常に良いモジュール**

***

### ③ Using GitHub Copilot Coding Agent

**難易度：★★☆〜★★★**

**理由**

*   自分で細かくコードを書かなくてよい
*   「計画 → 実行」をCopilotに委任できる
*   ここで *“人間はマネージャ”* という感覚が掴める

**学べること**

*   タスク分解をCopilotにどう指示するか
*   「最初に全部理解しなくていい」という安心感

👉 **スキル不足を補ってくれる“最強の味方”を体験できる**

***

### ④ Creating a Mini Game with GitHub Copilot

**難易度：★★★**

**理由**

*   Pythonコードは出てくるが、ロジックはCopilot任せでOK
*   成果物が視覚的・分かりやすい
*   「動いた！」という達成感が大きい

**注意点**

*   ゲームロジックを完璧に理解しようとしないこと
*   「Copilotが何を生成したかを見る」だけで十分

👉 **学習というより“体験型”なので、モチベーション維持に最適**

***

### ⑤ Integrate MCP with GitHub Copilot

**難易度：★★★★（最後に回すのがベスト）**

**理由**

*   MCP は概念寄りで抽象度が高い
*   Axios / Issue / PR 連携など、文脈理解が必要
*   早くやると「？？？」になりがち

**なぜ最後か**

*   CLI / Agent / Advanced Features を一通り経験したあとだと  
    「ああ、これ全部つなぐための仕組みなんだ」と腑に落ちる

👉 **“Copilotを開発基盤に組み込む”理解用の総仕上げ**
